# 文献調査サマリ：アーキテクチャレビュー方法論

## 1. ATAM: Method for Architecture Evaluation

- 著者: Rick Kazman, Mark Klein, Paul Clements
- 発行: Software Engineering Institute, CMU/SEI-2000-TR-004, 2000
- URL: https://www.sei.cmu.edu/documents/629/2000_005_001_13706.pdf
- Skillに反映した点:
  - 9ステップの評価プロセス
  - ビジネスドライバから品質特性シナリオへ展開する流れ
  - Utility Tree
  - アーキテクチャアプローチの分析
  - リスク、非リスク、感度点、トレードオフ点の分類
  - 最終レポートに含める成果物

## 2. Scenario-Based Analysis of Software Architecture / SAAM

- 著者: Rick Kazman, Gregory Abowd, Len Bass, Paul Clements
- 発行: IEEE Software / Software Engineering Institute, 1996
- URL: https://www.sei.cmu.edu/documents/213/1996_019_001_29912.pdf
- Skillに反映した点:
  - シナリオでアーキテクチャを評価する考え方
  - 候補アーキテクチャの記述
  - 直接シナリオと間接シナリオの区別
  - 変更影響と変更コストの記録
  - シナリオ相互作用による分離関心の評価

## 3. Quality Attribute Workshops (QAWs), Third Edition

- 著者: Mario R. Barbacci, Robert J. Ellison, Anthony J. Lattanze, Judith A. Stafford, Charles B. Weinstock, William G. Wood
- 発行: Software Engineering Institute, CMU/SEI-2003-TR-016, 2003
- URL: https://www.sei.cmu.edu/documents/716/2003_005_001_14249.pdf
- Skillに反映した点:
  - アーキテクチャ完成前の品質特性抽出
  - ステークホルダー参加型のシナリオ生成
  - 生シナリオ、統合、優先順位付け、精緻化
  - 刺激、刺激源、環境、対象、応答、応答測定を含むシナリオ形式
  - QAWシナリオをATAMの種シナリオとして使う流れ

## 4. Evaluating Software Architectures: Methods and Case Studies

- 著者: Paul Clements, Rick Kazman, Mark Klein
- 発行: Addison-Wesley, 2002
- Skillに反映した点:
  - アーキテクチャ評価を実務に適用するためのレポート構成
  - ATAM、SAAM、関連方法を使い分ける考え方
  - ケーススタディに基づく成果物の粒度

## 5. A Survey on Software Architecture Analysis Methods

- 著者: Liliana Dobrica, Eila Niemelä
- 発行: IEEE Transactions on Software Engineering, 28(7), 638–653, 2002
- URL: https://cris.vtt.fi/en/publications/a-survey-on-software-architecture-analysis-methods/
- Skillに反映した点:
  - アーキテクチャ評価の目的を、品質要求が設計で扱われているかの確認とリスク識別として定義
  - 複数方法の比較と、目的に応じた方法選択の必要性

## 6. Integrating ATAM with CBAM

- 著者: Robert Nord, Mario R. Barbacci, Paul Clements, Rick Kazman, Mark Klein, Liam O'Brien, James Tomayko
- 発行: Software Engineering Institute, CMU/SEI-2003-TN-038, 2003
- URL: https://www.sei.cmu.edu/library/integrating-the-architecture-tradeoff-analysis-method-atam-with-the-cost-benefit-analysis-method-cbam/
- Skillに反映した点:
  - ATAMの技術的トレードオフ分析に、コスト、利益、優先度を加えて意思決定ロードマップ化する考え方

## 7. Applying ATAM as Part of Formal Software Architecture Review

- 著者: Christopher Byrnes, Ioannis Kyratzoglou
- 発行: MITRE, 2007
- URL: https://www.mitre.org/sites/default/files/pdf/07_0094.pdf
- Skillに反映した点:
  - 制約のあるレビューでは、ATAMを軽量化・ハイブリッド化して使う実務上の考え方
  - CDRなど公式レビュー文脈でのチェックリスト、質問、ドキュメントレビューの活用

## 8. Supporting architecture evaluation for ATAM scenarios with LLMs

- 著者: Rafael Capilla, J. Andrés Díaz-Pace, Yamid Ramírez, Jennifer Pérez, Vanessa Rodríguez-Horcajo
- 発行: arXiv, 2025
- URL: https://arxiv.org/abs/2506.00150
- Skillに反映した点:
  - LLMは品質シナリオ、リスク、感度点、トレードオフ候補の整理を支援できる
  - ただし、最終判断は入力資料、ステークホルダー確認、検証結果に基づく必要がある
